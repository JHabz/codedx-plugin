This field lets you choose which Code Dx branch you would like the analysis to run on. This does not need to match your repository or code branch name. By default, Code Dx will run an analysis on the DEFAULT branch.

You can use build/environment variables to construct the chosen branch for each analysis. For a git project on branch fix-parameters: ${GIT_BRANCH} would start an analysis for Code Dx branch fix-parameters.

If the entered branch does not exist, Code Dx will create it if the analysis is successful as long as the parent is specified. If the analysis fails, no new branch will be created or saved.
